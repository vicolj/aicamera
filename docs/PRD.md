# AI 婴儿看护摄像头 — 产品需求文档 (PRD)

> 版本：v0.1 · 状态：草案 · 目标：0–3 岁婴幼儿家庭看护

## 1. 产品定位

面向新手父母的**智能婴儿看护摄像头**：在保障隐私的前提下，提供实时看护、异常告警、睡眠记录，减少夜间反复起床查看的次数。

**核心价值**

| 价值 | 说明 |
|------|------|
| 安心 | 哭声、遮脸、异常动作及时推送 |
| 省心 | 自动生成睡眠/清醒片段，减少手动记录 |
| 可控 | 本地优先处理敏感画面，云端可选 |

## 2. 用户与场景

**目标用户**

- 主用户：0–3 岁婴幼儿家长（妈妈/爸爸）
- 次用户：祖辈（仅查看，无配置权限）

**核心场景**

1. **夜间看护**：宝宝独自在婴儿床，家长在隔壁房间睡觉
2. **白天远程**：家长上班，老人在家，需手机随时查看
3. **午睡记录**：自动统计睡眠时长，生成日报
4. **安全告警**：遮脸、长时间无动静、异常哭声、离开安全区域

## 4. 非功能需求

| 类别 | 指标 |
|------|------|
| 视频延迟 | 局域网 WebRTC P99 < 800ms；公网 P99 < 2s |
| 告警延迟 | 哭声/遮脸：检测 → 推送 < 3s |
| 可用性 | 设备在线率 > 99.5%（排除用户断网） |
| 隐私 | 默认本地 AI；上传仅事件片段；AES-256 传输 |
| 功耗 | 整机 < 5W（墙插常供电） |
| 工作温度 | 0–40°C（婴儿房环境） |
| 认证 | CE/FCC（硬件出口）；SRRC（国内 WiFi） |

## 5. 交互与告警策略

**告警分级**

| 级别 | 类型 | 行为 |
|------|------|------|
| 🔴 紧急 | 遮脸、持续尖锐哭 | 强推送 + 声音 + 可选电话（V2） |
| 🟠 重要 | 一般哭声、离开安全区 | 推送 + 事件片段 |
| 🟡 提示 | 翻身、醒来 | 静默通知 / 时间线记录 |
| ⚪ 信息 | 入睡、醒来统计 | 仅记录，不推送 |

**默认策略**：夜间 22:00–07:00 仅 🔴🟠 强推送；白天全部级别可配置。

## 6. 约束与假设

- 安装高度：1.5–2.5m，俯拍婴儿床，覆盖床面 > 80%
- 网络：2.4GHz WiFi 必支持（IoT 兼容性）；5GHz 可选
- 首版市场：国内为主，App 中文优先
- 单机成本目标（BOM）：量产 5000+ 时 ≤ ¥180

## 7. 验收标准（MVP）

1. 在标准婴儿床场景下，哭声检测召回率 ≥ 90%，误报 ≤ 3 次/夜
2. 遮脸检测：口鼻遮挡 > 5s 触发告警，漏检率 < 5%
3. 连续运行 72h 无崩溃、无内存泄漏导致的重启
4. App 配网成功率 > 95%（BLE 辅助配网）
5. 断网后本地录像与告警仍可用，恢复网络后补传事件
